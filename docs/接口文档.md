## 从PLM中获取物料号对应的物料附件是否在变更流程中的字段的接口

请求体：
```
[
  "110105148060","110105148062"
]
```
请求体中的每个字段为物料编码，可以是一个，也可以是多个。在从EBS中获取了原始数据之后，需要根据存在表格中的所有物料的物料编码调这个接口去获取每个物料编码对应的“是否存在变更字段”

响应：
```
{
  "traceId": null,
  "timestamp": 1782109092,
  "data": [
    {
      "itemNumber": "110105148052",
      "isChange": false
    },
    {
      "itemNumber": "110105148053",
      "isChange": false
    }
  ],
  "code": 0,
  "success": true,
  "message": null,
  "stackTrace": null
}
```

响应中的data中就是我们需要的每个物料编码和对应的是否存在变更字段的数据，有一个需要注意的就是isChange为false就表示不存在变更，存在表格中的值就为“N”，如果是true表示存在变更，存在表格中的值就为“Y”
