## 回传EBS-ID和是否机加中心可以做这两个数据回EBS

请求体：
```
{
    "P_BATCH_NUMBER": "AI20260622163533",
    "P_IFACE_CODE": "CUX_AI_MACH_DRW_RST",
    "P_REQUEST_DATA": "[{\"SEQ_ID\":1018953, \"ORG_CODE\":\"H06\"},{\"SEQ_ID\":1018954, \"ORG_CODE\":\"H06\"}]"
}
```

P_BATCH_NUMBER:可选，给个AI+时间戳
P_IFACE_CODE:接口编码，固定值
P_REQUEST_DATA ：请求报文，SEQ_ID对应的就是EBS-ID，ORG_CODE对应的是是否机加中心可以做

响应：
```
{
  "OutputParameters": {
    "@xmlns": "http://xmlns.oracle.com/apps/cux/rest/WS/invokefmsws/",
    "@xmlns:xsi": "http://www.w3.org/2001/XMLSchema-instance",
    "X_BATCH_NUMBER": "AI20260622163533",
    "X_RETURN_CODE": "E",
    "X_RETURN_MESG": "Cux_Ai_Iface_Pkg.Mach_Drw_Rst_Parse_Data:ORA-06550: line 1, column 25:\nPLS-00302: component 'MACH_DRW_RST_PARSE_DATA' must be declared,CODE:-6550 cnt:1",
    "X_RESPONSE_DATA": null
  }
}
```
X_BATCH_NUMBER :同P_BATCH_NUMBER
x_RETURN_CODE : 接口返回编码，S成功，E失败
x_RETURN_MESG: 接口返回错误信息
x_RESPONSE_DATA：如果不成功，会把异常数据放在X_RESPONSE_DATA中原样返回

还有一点，就是在一个批次中点击回传到EBS的时候，应该要批量回传，现在这个接口也是能够满足的。同时，当有异常的时候
会有重新回传到EBS这个按钮，这个按钮就是重新回传单条的数据。
