# 004 去掉挂起异常默认原因 + 作业页自动暂存

- 状态：已完成
- 日期：2026-06-01
- 关联提交：（待提交）

## 需求 / 改动

1. 点「挂起异常」时，原因输入框默认预填了"图纸缺失"，希望去掉这个默认值，由操作员自己写。
2. 批次作业时，填写内容或挂起异常后要手点「暂存」才落盘，不方便。希望填了一条数据后能自动暂存。

## 方案

1. 去掉硬编码默认原因：原因输入框不再预填"图纸缺失"（仅当该行已有原因时回填，用于复改）。仍沿用"原因留空则不挂起"。
2. 防抖自动暂存：编辑待填列 / 挂起 / 撤销异常后，标脏并防抖 ~800ms（连续输入合并为一次写盘，避免每个键都写 xlsx）落盘。
   - 信息条上轻量提示"已自动暂存 · HH:mm:ss"，不弹 Toast 干扰填写。
   - 离开（返回按钮）/ 导航切走 / 关闭页面时把未落盘改动刷回（`Back` 先 flush 再导航；`Dispose` 尽力写回）。
   - 自动/手动暂存、刷盘共用一把 `SemaphoreSlim` 串行化，避免并发写同一 xlsx。
   - 已处理批次只读，不触发自动暂存。
   - 「暂存」按钮保留（手动立即落盘 + 成功 Toast，作为自动暂存的兜底与显式入口）。

## 实现

- `src/TZHJ.App/ViewModels/BatchWorkViewModel.cs`：
  - `MarkException`：默认原因由 `"图纸缺失"` 改为 `null`（无预填）。
  - 行编辑回调由 `Recompute` 改为 `OnRowChanged`（= `Recompute` + `ScheduleAutoSave`）；`MarkException`/`Restore` 末尾加 `ScheduleAutoSave`。
  - 新增自动暂存机制：`ScheduleAutoSave`（防抖，复用 `Application.Current.Dispatcher.InvokeAsync` + `Task.Delay` + `CancellationTokenSource`，与 `BatchListViewModel` 去抖同一套路）、`AutoSaveAsync`、`FlushAutoSaveAsync`、`SaveCoreAsync`（`SemaphoreSlim` 串行）、`_dirty` 脏标、`AutoSaveHint` 属性；`Back` 改 async 先 flush；`override Dispose` 刷盘；手动 `Save` 改走 `SaveCoreAsync`。
- `src/TZHJ.App/Views/BatchWorkView.xaml`：信息条加 `AutoSaveHint` 提示文本。

不变：提交闸门、整批回传、行状态机、其余页面。
