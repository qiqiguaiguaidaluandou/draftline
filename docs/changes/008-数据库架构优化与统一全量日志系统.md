# 变更 #008：数据库架构优化与统一全量日志系统

> **状态**：方案已定，待实施
> **日期**：2026-06-16

## 1. 优化背景
随着系统演进到“远程优先（Remote-First）”镜像架构，现有的 5 张数据库表在数据一致性、可追溯性以及加载性能上存在优化空间。特别是审计与操作日志的分离，不利于进行全链路的行为分析。

## 2. 优化目标
1. **全量日志化**：取消独立的审计表和行为日志表，构建统一的活动流水线。
2. **性能加速**：在批次表中固化元数据，消除客户端加载列表时对物理 Excel 的实时扫描依赖。
3. **闭环异常管理**：将异常池从“物理删除”模式升级为“状态机”模式，保留所有处理历史。
4. **强一致性约束**：引入数据库外键约束，确保物理删除批次时自动清理关联异常。

## 3. 表结构变更方案

### 3.1 统一活动日志 `ActivityLogs` (新增)
*取代现有的 `AuditRecords` 和 `OperationLogs`。*

| 字段名 | 类型 | 说明 |
| :--- | :--- | :--- |
| **Id** | BigInt (PK) | 自增主键 |
| **Timestamp** | DateTime | 发生时间（UTC） |
| **EmployeeId** | String | 执行人账号/工号 |
| **Action** | String | 动作类型（Login, Submit, UpdateRow, Suspend, Resolve, Ingest, Cleanup） |
| **Flow** | Enum | 业务流程（Pricing / DrawingSelection） |
| **GroupName** | String | 所属产品线组 |
| **BatchId** | String | 关联的批次 ID |
| **ImpactCount** | Int | 受影响行数（例如：回传了 X 行，清理了 Y 个批次） |
| **Status** | String | 结果状态（Success / Failed） |
| **Payload** | JSON/Text | 关键快照信息（如：EBS 返回的审计号、报错信息、修改前后的值） |
| **ClientIp** | String | 客户端 IP |

### 3.2 增强批次注册 `BatchRegistries` (修改)
| 字段名 | 类型 | 变更说明 |
| :--- | :--- | :--- |
| **TotalRows** | Int | **[新增]** 该批次物料总行数。采集时计算，用于列表页秒开统计。 |
| **CreatedAt** | DateTime | **[新增]** 该批次最初进入云端的时间。 |
| **LastModified** | DateTime | 记录最后一次行更新或状态迁移的时间。 |

### 3.3 升级异常池 `Exceptions` (修改)
| 字段名 | 类型 | 变更说明 |
| :--- | :--- | :--- |
| **Status** | Enum | **[新增]** 状态（Exception/Pending -> Uploaded/Resolved）。不再直接物理删除。 |
| **ResolvedAt** | DateTime | **[新增]** 处理完成时间。 |
| **ResolvedBy** | String | **[新增]** 处理人账号。 |
| **ResolutionAuditId** | String | **[新增]** 补处理成功后关联的 `ActivityLogs.Id` 或审计标识。 |

## 4. 业务逻辑对齐

### 4.1 数据采集 (Data Ingestion)
1. **统计注入**：`DataIngestionService` 在向磁盘写 Excel 的同时，将行数存入 `BatchRegistries.TotalRows`。
2. **系统日志**：每次自动采集后，向 `ActivityLogs` 记入一条 `Action=Ingest` 的记录。

### 4.2 镜像同步 (Mirror Sync)
1. **轻量化清单**：`/api/sync/catalog` 直接返回 `TotalRows`，客户端列表显示的 `0 / 5` 进度不再需要解压/读取物理 Excel，实现瞬间加载。
2. **历史追溯**：`/api/sync/exceptions` 默认只返回 `Status=Open` 的异常，但保留查阅已处理历史的能力。

### 4.3 行为审计 (Unified Audit)
1. **登录**：记录 IP 和时间。
2. **回传**：在 `ActivityLogs` 中记录受影响行数和 Payload（含外部系统回显）。
3. **数据清理**：自动清理 15/30 天过期数据时，记录清理了多少个文件夹，作为系统自维护证据。

## 5. 迁移计划
1. **模型定义更新**：修改 `Draftline.Gateway.Stores` 下的实体类。
2. **DbContext 重构**：配置新的联合主键、外键及索引。
3. **生成迁移脚本**：执行 `dotnet ef migrations add OptimizeSchemaAndLogs`。
4. **逻辑适配**：批量重构 `ApiEndpoints` 中的日志写入代码。
5. **数据平滑迁移**：将旧的 `AuditRecords` 数据脚本迁移至 `ActivityLogs`（可选）。
