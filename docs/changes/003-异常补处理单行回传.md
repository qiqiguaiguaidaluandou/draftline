# 003 异常待跟进：补处理 + 单行回传

- 状态：已完成
- 日期：2026-06-01
- 关联提交：b527538

## 需求 / 改动

异常待跟进页改前只读列示异常数据，异常进池后无出口、无法解决。希望：点某条异常 → 显示该行全部字段 → 重填需填字段 → 用一个"上传"按钮**单独回传这一行** → 回传成功后，把它在**来源批次（已处理）表单中的行状态由"挂起异常"改为"已上传"**。

## 方案

链路：异常待跟进页 → 点某行「补处理」→ 进入该行**补处理页**（竖排显示全字段，待填列可编辑、其余只读，可打开本行图纸）→ 重填 → 「上传」→ 成功后：①单行回传；②来源批次那行 `挂起异常`→`已上传`；③该条移出异常池；④返回异常列表（已不含它）。

关键决策与依据：

- **字段从来源批次读**，不在异常池冗余存全字段。挂起异常的行其实一直留在 `已处理/<来源批次>/表格.xlsx` + manifest 里（`Status=Exception`），图纸也在该文件夹。异常池 `exceptions.json` 只是索引（料号/名称/来源批次/原因/时间）。按 `来源批次 + RowKey` 用 `GetBatchAsync` 把整行读出即得全字段——补回传与"改行状态"天然是同一个对象，不用两头对账。
- **回传成功后异常直接移出池**（选项 A），不留"已解决"台账。该页定位即"待跟进"，解决了就该离开；要追溯的话来源批次那行已是 `Uploaded`，有据可查。
- **UI 用独立页面**而非弹窗：挑图 16 字段弹窗会挤；独立页与现有作业页导航模型一致，放得下字段+图纸+上传按钮。
- **来源批次可能已不在**（已处理有保留期，网关配置默认 30 天）：读不到则提示"来源批次已不存在，无法补回传"、禁用上传。

## 实现

1. `src/Draftline.Core/Contracts/ILocalBatchStore.cs` + `src/Draftline.Infrastructure/Storage/LocalBatchStore.cs`：新增 `RemoveExceptionAsync(flow, employeeId, sourceBatch, rowKey)`——读 `exceptions.json`、按"来源批次+行键"删匹配项、写回（无匹配则不动文件）。
2. 新增 `src/Draftline.App/ViewModels/ExceptionResolveViewModel.cs`：`GetBatchAsync(flow, 工号, Done, 来源批次)` 取该行 → 包成 `RowViewModel`（复用网格那套 `[key]` 绑定 + 待填列自动判定）→ 调 `Row.Restore()` 解除挂起态以便编辑判定；`Upload` 命令校验行已 `Done` → `SubmitBatchAsync`（单行，BatchKey/窗口取来源批次）→ 成功后 `Row.MarkUploaded()` + `SaveBatchAsync` 写回来源批次 + `RemoveExceptionAsync` 移出池 → 返回异常列表。`CanResolve=false`（来源批次/行不存在或无回传权限）时只读提示、禁用上传。
3. 新增 `src/Draftline.App/Views/ExceptionResolveView.xaml(.cs)`：竖排全字段表单（字段行由代码后置按 schema 生成：待填文本→TextBox、待填下拉→ComboBox、其余只读文本；`FormHost.DataContext=Row`，绑 `[key]`），含返回/打开来源批次/上传按钮 + 闸门提示。`App.xaml` 加 `ExceptionResolveViewModel→ExceptionResolveView` 模板映射。
4. `src/Draftline.App/Services/INavigationService.cs`：加 `ToExceptionResolve(flow, ExceptionItem)`。
5. `src/Draftline.App/ViewModels/ExceptionPoolViewModel.cs`：注入 `INavigationService`，加 `ResolveCommand`；`src/Draftline.App/Views/ExceptionPoolView.xaml` 异常行加「补处理」按钮。
6. `tests/Draftline.Tests/Sync/Fakes.cs`：`FakeLocalBatchStore` 补上 `RemoveExceptionAsync`（未用到，抛 NotSupported）。

不变：挂起异常的产生逻辑、整批回传、批次列表/作业页、权限、整体风格。
