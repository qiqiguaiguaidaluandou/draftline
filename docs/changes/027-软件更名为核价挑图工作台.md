# 027 客户端更名为「核价挑图工作台」+ 换应用图标

- 状态：**已实现**
- 日期：2026-07-13
- 关联提交：ddb1d62（实现）

> 结论：客户端显示名从「图纸核价 / 挑图纸客户端」（及应用内「图纸核价 / 机加工挑图系统」）
> 统一改为 **「核价挑图工作台」**。只改用户可见的显示文案；**内部程序集名 `Draftline.App` 不动**
> （它是 ClickOnce 部署入口 `Draftline.App.application` 与 `/draftline` URL 的一部分，改了会断更新链路）。

## 改动的 6 处显示名

| 位置 | 作用 | 改后 |
|---|---|---|
| `Draftline.App.csproj` `<Product>` | 装到本地后「添加/删除程序」「开始菜单」显示名 | 核价挑图工作台 |
| `Draftline.App.csproj` `<AssemblyTitle>` | exe 文件说明 | 核价挑图工作台 |
| `ClickOnceProfile.pubxml` `<ProductName>` | ClickOnce 安装/快捷方式名 | 核价挑图工作台 |
| `LoginWindow.xaml` Title + 大标题 | 登录窗 | 核价挑图工作台（标题栏加「— 登录」） |
| `ShellWindow.xaml` Title + 顶栏标题 | 主窗口 | 核价挑图工作台 |

发布者/组织名：`<Company>` / `pubxml <PublisherName>` 由「图纸核价系统」→ **「核价挑图系统」**（装机后显示的“发布者”）。

保持不变：`AssemblyName`/`RootNamespace` = `Draftline.App`（内部标识）；URL `/draftline`。

**刻意不动**：导航/标题/异常池里那些「图纸核价」「挑图纸」——它们是**两个流程**的名字（核价流程、挑图流程），
不是软件名，改成「核价挑图」会与并列的「挑图纸」冲突。`LocalFolders.Pricing="图纸核价"` 是本地存储文件夹名，
改了会让老客户端已下载的文件夹对不上（数据孤儿），一并保留。EBS/排程等代码注释里的「图纸核价」指核价流程/接口，非软件名，未动。

## 应用图标（换图标）

用户提供的 `docs/核价挑图软件图标.png`（2048² 蓝图+铅笔+路框设计）带两处杂物，已用 PIL 清理：
- **右下角「通义万相」水印**：因压在弯道路框上，纯填色会毁弧线；改用**左下角对称区域水平镜像重建右下角**
  （路框左右对称），边缘 60px 羽化，接缝不可见、路面弧线连续。
- **底部「◆机加核价◆」文字带**：按行在左右边界间**水平插值**填深蓝底（贴合横向渐变，无方框缝）。

清理后母图存 `docs/核价挑图软件图标-无水印.png`（原图保留）。据此生成资源并接入：
- `Assets/app-icon.ico`（多分辨率 16/32/48/64/128/256）→ csproj `<ApplicationIcon>`（exe/任务栏/添加删除程序）。
- `Assets/app-icon.png`（256，`Resource`）→ `LoginWindow`/`ShellWindow` 的 `Icon`（标题栏图标）。

## 说明

- 老客户端更新到本版后，快捷方式/程序列表里的名字会随之变为「核价挑图工作台」、图标同步更新。
- WPF 客户端构建 0 error 0 warning（Linux 用 `-p:EnableWindowsTargeting=true` 仅编译；图标真机需 Windows 运行确认显示）。
