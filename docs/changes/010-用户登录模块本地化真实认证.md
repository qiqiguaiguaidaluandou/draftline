## 需求 / 改动

明确不再接入 DHR/SSO：**账号与密码改由本系统管理员维护**（管理员建用户、配权限）。此前认证层全是占位，存在真实安全隐患，本次将其落地为真实本地认证：

改前现状（占位）：
1. **`FakeAuthService`**：任意非空工号即可登录，密码除魔法串 `"fail"` 外全部忽略（`Auth/IAuthService.cs`）。
2. **`FakeTokenService`**：令牌形如 `tzhj:{工号}:{guid}`，**无签名、可任意伪造、永不过期**（`Auth/ITokenService.cs`）。任何人构造 `tzhj:别人工号:x` 即可冒充。
3. **无用户表**：凭证根本不落库；`/api/auth/login` 在开发环境对任意工号 **Auto-Seed** 两条流程权限，架空了 doc ⑧ 的 Deny-All 白名单（`Endpoints/ApiEndpoints.cs`）。
4. **无登录限流/锁定**，可暴力猜解。
5. **改密窗口是空壳**，文案写"密码由 DHR 统一管理"，不接后端（`Views/ChangePasswordWindow`）。
6. **客户端**：硬编码默认工号 `108645`、无输入校验（`LoginViewModel`）。

## 方案

一次「登录模块本地化真实认证」变更，**不依赖任何外部接口**，可在 Linux 编译+测试：

1. **用户与凭证落库**：新增 `AppUser` 实体（工号唯一、姓名/部门/岗位、密码哈希、启用、管理员、强制改密、失败计数、锁定到期、时间戳）。权限继续用现有 `UserPermissions(工号,流程,组)` 表（不另起炉灶）。
2. **密码哈希**：用 ASP.NET Core 自带 `PasswordHasher<T>`（PBKDF2-HMAC-SHA256，自带随机盐与算法版本），封装为 `IPasswordService`。不引入 Argon2/bcrypt 外部库。
3. **真实 JWT**：`JwtTokenService` 替换 `FakeTokenService`，HMAC-SHA256 签名 + `exp` 过期 + 颁发者/受众校验；密钥/有效期来自 `Jwt` 配置段（密钥放 gitignore 的 `appsettings.local.json`）。`ITokenService` 接口形状不变，`TokenEndpointFilter` 不动。
4. **认证服务**：`DbAuthService` 校验本地凭证：账号不存在与密码错误返回**同一提示**（防工号枚举）；连续失败 5 次锁定 15 分钟；停用账号拒登；成功按 `UserPermissions` 组装 `AllowedFlows`、签发 JWT。
5. **登录端点重写**：去掉 Auto-Seed；成功/失败都记登录审计。
6. **本人改密**：新增 `POST /api/auth/change-password`（需令牌，工号以令牌为准），校验旧密码、新密码 ≥8 位且不同于旧密码。
7. **管理端**：新增 `/api/admin/*`（`AdminEndpointFilter` 实时查库要求"启用中的管理员"）：列用户 / 建用户 / 重置密码 / 启停用 / 覆盖式配权限。所有写操作记管理审计。
8. **引导管理员**：启动时若用户表为空且配置了 `Admin` 段，则种子一个管理员（首登强制改密），解决"有库无人能登录"。
9. **客户端加固**：去掉硬编码工号、登录前输入校验；改密窗口接真后端 + 8 位策略 + 修正文案；`AuthResult.MustChangePassword` 透传到会话，首登强制改密提示。

取舍：
- **进程内锁定/限流**够用（单网关实例，见 `方案设计.md §10`）；多实例需共享存储，留待。
- **令牌仍在客户端内存**（不落盘）：JWT 已带过期，重启重登更安全，故有意不做 DPAPI 持久化。
- 功能级 `CanOperate/CanSubmit` 暂恒为真（无细粒度功能角色），后续可按 `IsAdmin`/角色细化。
