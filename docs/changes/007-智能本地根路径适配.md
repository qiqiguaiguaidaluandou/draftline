# 007 智能本地根路径适配

- 状态：待实现
- 日期：2026-06-10
- 关联提交：<git 短哈希>

## 需求 / 改动

解决操作员机器环境差异化导致的本地路径配置困难问题：
- **现状**：`appsettings.json` 中硬编码了 `LocalRoot`（如 `C:\Users\108645\Desktop\TZHJ_Data`），这在其他操作员机器上会导致路径无效，且在 GitHub 上传时会泄露开发者隐私。
- **痛点**：每个操作员都需要手动修改配置文件才能运行，门槛较高且容易出错。
- **目标**：实现“开箱即用”。软件启动时应能自动识别环境并生成合法的默认路径，同时支持手动覆盖。

## 方案

选择 **“强制系统标准路径”** 方案：
- **完全取消配置化**：不再从 `appsettings.json` 或环境变量读取 `LocalRoot`。
- **硬编码锁定**：程序启动时，强制将根目录设为当前用户的 **“我的文档”** (My Documents) 下的 `TZHJ_Data` 文件夹。
- **优点**：
  - **极致简单**：消除了所有关于路径配置的沟通成本。
  - **运维一致性**：所有操作员的路径逻辑完全一致，方便远程支持。
  - **隐私与安全**：不再需要担心路径中包含敏感信息被上传。

## 实现

**客户端 (TZHJ.App)**

- `App.xaml.cs`：修改 `OnStartup` 逻辑，在绑定 `HttpOptions` 后，强制覆盖 `LocalRoot` 属性为 `Environment.SpecialFolder.MyDocuments` 拼接 `TZHJ_Data`。

**后端 (TZHJ.Gateway)**

- `Stores/InMemoryConfigStore.cs`：同步调整。后端下发给客户端的 `ClientConfig.LocalRoot` 也固定为此逻辑，确保客户端逻辑闭环。

**配置文件**

- 移除 `src/TZHJ.App/appsettings.json` 和 `src/TZHJ.Gateway/appsettings.json` 中的 `LocalRoot` 配置项。


## 验证

1.  **路径锁定验证**：
    *   即使在 `appsettings.local.json` 中配置了其他路径，启动程序后，通过“设置”页或日志确认根目录依然指向“我的文档\TZHJ_Data”。
2.  **自动创建验证**：
    *   删除“我的文档”下的 `TZHJ_Data` 文件夹，启动程序，确认文件夹被自动重新创建。
3.  **零配置验证**：
    *   在一台全新的机器上安装，不提供任何配置文件，确认程序依然能正常初始化并工作。
