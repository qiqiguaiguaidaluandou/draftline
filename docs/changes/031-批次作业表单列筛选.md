- 日期：2026-07-16

## 需求 / 改动

批次作业表单（`BatchWorkView`）此前是一张只能上下滚的可编辑网格，行多时不好定位。业务希望像 Excel 那样**每列表头点 ▼ 筛选**，快速缩小到关心的行。

只做**筛选**（不含排序）。按流程指定可筛选列：

- **核价（Pricing）**：物料编码、型号、名称、变更状态（"当前是否存在变更"）
- **挑图（DrawingSelection）**：产品线、申请人名称、项目

## 方案

- **只影响显示、不动数据**：把网格数据源 `Rows` 挂到 `ICollectionView.Filter` 上做可见性过滤，`Rows` 集合本身与其中每行的取值都不改。
- **提交闸门安全**：`PendingCount` / `DoneCount` / `CanSubmit` 等统计本就基于 `Rows` 全量（非视图）计算，筛选后维持不变——**杜绝"筛选隐藏了未处理行却能点回传"的事故**，故 VM 统计口径无需任何改动。
- **可筛选列是业务规则**，放在 VM 层（`FilterableKeys`），视图只按此集合决定"哪些列挂 ▼"，保持视图通用。
- **候选值实时取**：每次点开 ▼ 都按当前 `Rows` 重新去重取值，能反映填写 / 重新获取图纸等带来的变化。
- **交互取舍**：筛选列关闭表头点击排序（`CanUserSort=false`），避免与 ▼ 混淆（需求为"只要筛选"）；其余列排序行为不变。
- **不引入第三方库**：自建轻量筛选控件，与现有 FluentTheme 视觉一致（复用 `Accent`/`PanelBg`/`BtnPrimary` 等资源）。

筛选面板内含：搜索框、`(全选)` 三态勾选、值勾选列表、`确定` / `清除筛选`。空值显示为 `(空)`；全勾等于不筛选；处于筛选态时该列 ▼ 变主题蓝并加粗、悬停提示"已筛选"。

## 实现

**新增**
- `src/Draftline.App/Controls/FilterOption.cs`：下拉里的一个候选值（`Raw` 归一化取值 / `Display` 显示文本 / `IsChecked`）。
- `src/Draftline.App/Controls/ColumnFilterHeader.xaml(.cs)`：表头筛选控件（"列名 + ▼" + 弹出面板）。对外暴露 `Title`、`DistinctValuesProvider`（下拉打开时实时取列值）、`ApplyRequested`（回调触发视图刷新）、`Matches(raw)`（供行谓词逐列判定）。筛选态：`_active` 为 `null` 表示未筛选放行，否则为"允许通过的取值集合"（全不勾即空集、隐藏所有行，与 Excel 一致）。

**改动**
- `src/Draftline.App/ViewModels/BatchWorkViewModel.cs`：新增 `IReadOnlySet<string> FilterableKeys`，构造时按 `FlowType` 用 `FieldSchemas.PricingKeys` / `DrawingKeys` 常量填充（核价 4 列 / 挑图 3 列）。
- `src/Draftline.App/Views/BatchWorkView.xaml.cs`：
  - `BuildColumns` 对命中 `FilterableKeys` 的列调用 `AttachFilter`——把列 `Header` 换成 `ColumnFilterHeader`、关闭该列排序、登记到 `_filters`。
  - `OnLoaded` 建列后取 `CollectionViewSource.GetDefaultView(vm.Rows)` 并设 `Filter = RowPassesFilters`（逐列 `header.Matches(r[key])` 求与）。
  - `ApplyFilters` = `_view.Refresh()`。

`BatchWorkView.xaml` 未改（排序在代码后置按列关闭；网格数据源不变）。
