## 需求 / 改动

`docs/接口文档.md` 改为给出 PLM 的两个接口。EBS 只给"基础数据"，图纸文件与"是否在变更流程中"来自 PLM——在 EBS 取到原始数据后，**带着所有行的物料编码**调 PLM 富化：

- **变更状态接口**：请求体是裸的物料编码 JSON 数组 `["110105148060","110105148062"]`（一个或多个）；响应 `data[]{itemNumber, isChange}`。`isChange=false`→表格存 `"N"`，`true`→`"Y"`。
- **图纸附件接口**：同样 POST 裸物料编码数组；响应 `data[]{itemNumber, itemName, files[]{fileName, fileType, fileSize, fileStr}}`。`fileStr` 是 MinIO 预签名 URL（含 AWS 签名、`X-Amz-Expires=604800`≈7天），需再 **GET 该 URL 下载真正的字节**。图纸存在对应批次文件夹下、与表格同一文件夹。

013 时 PLM 文档未到，`EbsPlmSource` 把 `hasChange` 留空、`OpenDrawingAsync` 返回 null。本次补齐。

## 方案

- **接入点 = 取数阶段的富化**：现有架构在 ingestion 时就把图纸字节随表格一起落进批次文件夹（`PersistGroupAsync` → `SaveBatchAsync`），正好对应文档"图纸存批次文件夹下、与表格同一文件夹"。所以 PLM 调用全部发生在 `EbsPlmSource.FetchRowsAsync` 末尾，对 EBS 映射出的行做富化——填 `hasChange`、把下载好的图纸字节挂到 `SourceRow.Drawings`。上层（采集服务、落盘、客户端、字段 schema）一律不动。`OpenDrawingAsync` 维持返回 null：图纸已在取数阶段落盘、由磁盘流式服务，无控制器走此方法。
- **鉴权复用 EBS**：PLM 与 EBS 用同一套 JWT（同 `Ebs:JwtSecret`/`JwtIssuer`/`AuthScheme`），直接复用 `EbsTokenProvider`，不另配密钥。POST body 就是裸的 JSON 字符串数组、`Content-Type: application/json`。
- **图纸下载无需鉴权**：`fileStr` 自带 AWS 签名，GET 时**不带** `Authorization` 头。
- **两个 flow 都富化**：核价（图纸核价）与挑图（机加中心挑图）的行都填 `hasChange`、都挂图纸（与 Fake 现状一致）。两个 schema 的 `materialCode`/`hasChange` 键名相同。
- **URL 留空 = 跳过**：`Plm:ChangeUrl` 空 → 跳过变更富化（`hasChange` 留空）；`Plm:DrawingUrl` 空 → 跳过图纸下载（图纸标"缺失"）。便于在 PLM 地址未就绪时先单独验证 EBS。
- **分批**：物料过多时按 `Plm:BatchSize`（默认 200）分批 POST。
- **韧性**：PLM 接口整体调用失败（HTTP 非 2xx / `success=false`）→ 向上抛异常，使本批不登记、下个 5 分钟轮询重采（符合"没采到就啥都不留"的自愈设计）；**单个料号**在 PLM 未返回 → 该行 `hasChange` 留空、无图纸，不连累其余行；**单个图纸文件**下载失败/超大小上限 → 跳过该文件并告警，不阻断整批。
- **料号在 PLM 未返回时的 `hasChange`**：留空（表示"未知"，既非 Y 也非 N）。

## 实现

**新增**（`src/TZHJ.Gateway/AntiCorruption/`）：
- `PlmOptions.cs`：`Plm` 配置段——`ChangeUrl`/`DrawingUrl`（占位空）、`BatchSize`(200)、`MaxDrawingBytes`(50MB)。
- `PlmClient.cs`：
  - `FetchChangeFlagsAsync(codes)` → 分批 POST → 解析 `data[]{itemNumber,isChange}` → `itemNumber→bool` 字典。
  - `FetchDrawingsAsync(codes)` → 分批 POST → `data[]{itemNumber,files[]}` → 逐个 GET `fileStr` 下载字节（带大小上限与异常兜底）→ `itemNumber→[(fileName,content)]`。
  - 外层统一校验 `success`、解析 `data`；鉴权复用 `EbsTokenProvider`+`Ebs:AuthScheme`。

**改动**：
- `Gateway/AntiCorruption/EbsPlmSource.cs`：注入 `PlmClient`；`FetchRowsAsync` 末尾调新增的 `EnrichWithPlmAsync`（收集去重物料编码 → 调两接口 → 回填 `hasChange`、追加 `SourceRow.Drawings`）。`MapPricing`/`MapDrawing` 里 `hasChange` 初值注释由 `TODO(PLM)` 改为"由 PLM 富化填 Y/N"。`OpenDrawingAsync` 改为表达式体、注释说明走磁盘。
- `Gateway/Program.cs`：`Ebs:Enabled` 为真时，绑定 `PlmOptions`、`AddHttpClient<PlmClient>()`、再 `AddHttpClient<IEbsPlmSource, EbsPlmSource>()`（`EbsPlmSource` 由 DI 注入 `PlmClient`）。
- `Gateway/appsettings.json`：加 `Plm` 段（占位，URL 空）。
