# 变更记录（changes）说明与模板

本目录沉淀**小需求 / 小改动**：每有一处需求或调整，就形成一份文档，记清"改什么、怎么改、怎么落地的"，方便回溯与交接。

> 大块设计仍以 `docs/方案设计.md` / `开发文档-①②` / `路线图.md` 为权威；本目录只装零散的小变更。

## 约定

- 一处变更一份文件，命名 `NNN-简短标题.md`（`NNN` 三位顺序号，从已有最大号 +1；本文件占 000）。
- 文件用下面的模板，至少写齐：**需求/改动、方案、实现**三段。
- 状态取值：`待实现` / `进行中` / `已完成`。已完成的关联上提交哈希。
- 日期用绝对日期（如 2026-06-01），别写"今天/上周"。

## 模板

```markdown
# NNN 标题

- 状态：待实现 / 进行中 / 已完成
- 日期：YYYY-MM-DD
- 关联提交：<git 短哈希>（已完成时填）

## 需求 / 改动
（要解决什么、改前是什么样、为什么要改）

## 方案
（怎么做；关键决策与取舍；为什么这么选）

## 实现
（实际落地：改了哪些文件、关键改动点；未实现时写"待实现"）

## 验证
（编译/测试结果；WPF 真机待验项）

## 备注 / 待办
（遗留问题、接真接口时要补的部分等；没有可省略）
```
