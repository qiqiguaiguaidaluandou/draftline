# 035 本地数据存放路径可自选（不再固定 C 盘）

- 状态：**实现完成，待运行态验证**
- 日期：2026-07-17
- 关联提交：`__PENDING__`

## 需求

本地文件此前被固定存放在「我的文档\Draftline_Data」（通常在 C 盘），用户无法更改。
需要让用户在本机自行选择存放路径；更改后原有数据要一起迁到新位置，且软件里各处
「打开文件夹」也要跟随新路径。

## 现状（改动前）

`App.xaml.cs` 把本地根写死为「我的文档\Draftline_Data」，注释明确「强制标准化，不再受配置文件影响」。
全项目统一从单例 `LocalStorageOptions.Root` 取根（`LocalPaths` 全靠传入 root 拼路径），
所以改这一处即全局生效——架构上友好，缺的只是「让用户选 + 持久化 + 迁移」。

## 方案

### 决策（与用户确认）
- 更改路径后，旧目录数据**自动迁移**到新位置。
- 入口：**设置页 + 首次运行**都提供。
- 生效方式：**重启后生效**（迁移放到重启后、任何组件用到路径之前做，避开运行中的同步与文件句柄，最安全）。

### 关键工程点
- 用户选定的路径持久化在 **`%AppData%\Draftline\client-settings.json`**，
  **不能**放程序目录：ClickOnce 程序目录带版本号、每次更新即换，写那里会丢。
- 启动解析顺序（`App.ResolveLocalRoot`，均在 DI/登录之前）：
  1. 首次运行（无设置文件）→ 弹窗问是否自定义位置；选是则弹系统文件夹对话框。无论选与否都落盘，避免每次都问。
  2. 有「待迁移」标记 → 把旧根整体迁到新根，成功后清标记；**失败则退回旧根继续用**（不丢数据、不把用户困在空目录）。
  3. 返回最终根 → 赋给 `http.LocalRoot` → DI 建 `LocalStorageOptions { Root = ... }`。
- 迁移用 `DirectoryMigrator.Move`：同卷 `Directory.Move` 秒改名；跨卷（C→D，Move 抛 IOException）退回递归复制 + 删源；目标已存在则合并、同名覆盖；源缺失/同路径静默返回（幂等）。
- 设置页「更改」：选父目录 → 拼成 `<所选>\Draftline_Data` → 确认 → 写「新根 + 待迁移=旧根」→ 提示重启 → `App.Restart()`。

### 各处「打开文件夹」是否跟随（确认，无需额外改动）
全部入口均从同一个 `LocalStorageOptions.Root` 推算，重启后统一切换：

| 入口 | 路径来源 |
|---|---|
| 设置页「打开」 | `storage.Root` |
| 批次列表「打开位置文件夹」 | `LocalPaths.LocalLocationRoot(storage.Root, …)` |
| 异常池「打开」 | `LocalPaths.LocalExceptionPoolRoot(storage.Root, …)` |
| 批次作业「打开文件夹」 | `_batch.FolderPath` ← `LocalPaths.LocalBatchDir(Root, …)` |
| 异常处理「打开」 | `_sourceBatch.FolderPath` ← 同上 |

因全程只有一个根变量、运行中不被改（只经设置文件 + 重启），不存在「一半新一半旧」的状态；
且因自动迁移，重启后按钮打开的是真实、有内容的新目录。

## 实现

新增（均在 Infrastructure，可测；测试项目不引用 WPF 的 App）：
- `Options/ClientSettings.cs`：本机设置模型（`LocalRoot`、`PendingMoveFrom`）。
- `Storage/ClientSettingsStore.cs`（+ `IClientSettingsStore`）：读写 `%AppData%\Draftline\client-settings.json`，原子替换。
- `Storage/DirectoryMigrator.cs`：跨卷安全的目录迁移（合并覆盖、幂等）。
- `Storage/LocalRootResolver.cs`：默认根与 `Draftline_Data` 命名约定收敛一处。

改动：
- `App.xaml.cs`：反转「强制固定 C 盘」，改为 `ResolveLocalRoot`（首次提示 + 待迁移执行）；新增 `Restart()`。
- `Services/IExplorerService.cs`：加 `PickFolder`（用 .NET 8 `Microsoft.Win32.OpenFolderDialog`，无新依赖）。
- `ViewModels/SettingsViewModel.cs`：`ChangeRootCommand`（选路径 → 确认 → 记录 → 重启）。
- `Views/SettingsView.xaml`：数据根一行加「更改」按钮。
- `DependencyInjection.cs`：注册 `IClientSettingsStore`。
- `ViewModels/LoginViewModel.cs`：更新过时注释（本地根来源）。

测试：
- `tests/Draftline.Tests/Storage/DirectoryMigratorTests.cs`：迁移并删源、合并覆盖、源缺失/同路径幂等（3 例）。
