## 接口对接（来自 docs/接口文档.md）

**请求**（POST，鉴权复用 EBS 的 JWT，同取数）：

```json
{
  "P_BATCH_NUMBER": "AI20260713xxxxxx",
  "P_IFACE_CODE": "CUX_AI_MACH_DRW_RST",
  "P_REQUEST_DATA": "[{\"SEQ_ID\":1018953, \"ORG_CODE\":\"H06\"}, ...]"
}
```

- `P_BATCH_NUMBER`：可选，取 `AI` + 本地时间戳（`yyyyMMddHHmmss`）。
- `P_IFACE_CODE`：固定 `CUX_AI_MACH_DRW_RST`。
- `P_REQUEST_DATA`：**转义的 JSON 字符串**（不是嵌套对象），数组元素 `{ SEQ_ID, ORG_CODE }`。
  - `SEQ_ID` ← `DrawingKeys.EbsId`（挑图行主键 / EBS-ID），能转数字则发数字，否则发字符串。
  - `ORG_CODE` ← `DrawingKeys.CanMachine`（是否机加中心可以做，手填下拉 `H06`/`否`）：
    **`H06` 原样发；`否` → 映射为 `"N"`**（对方约定）。

**响应**（批次级判定，与 SRM 的逐行 `data[].remark` 不同）：

```json
{ "OutputParameters": { "X_BATCH_NUMBER": "...", "X_RETURN_CODE": "S|E", "X_RETURN_MESG": "...", "X_RESPONSE_DATA": null } }
```

- `X_RETURN_CODE=S` → 整批成功 → 所有行成功（`X_RETURN_MESG`/`X_RESPONSE_DATA` 均为 null，含 `@xmlns` 字段被忽略）。
- `X_RETURN_CODE=E` 且 `X_RETURN_MESG` **含「部分失败」**（如 `数据更新部分失败！`）→ **部分失败**：
  `X_RESPONSE_DATA` 里回来的 `SEQ_ID` 为失败行（转入异常池），其余行视为已落库成功，批次置 Done。
- `X_RETURN_CODE=E` 的**其余任何情形** → **抛异常**（整批回滚、未落库、批次不置 Done、客户端可重新回传）。
- **重要判据**：对方明确——**任何 `E` 的 `X_RESPONSE_DATA` 都会带上未成功的数据**，所以区分「部分失败(入池)」
  与「其他 E(整批回滚)」靠的是 **`X_RETURN_MESG` 是否为部分失败**，**不是**「`X_RESPONSE_DATA` 有没有数据」。
  代码用 `X_RETURN_MESG.Contains("部分失败")` 判定（对标点/前后缀更稳）。
- `X_RESPONSE_DATA` 解析**两种形态都容错**：① 转义 JSON 字符串（同 `P_REQUEST_DATA`，真实样例即此形态）；
  ② 直接是数组/对象。防御：声明部分失败却解析不出失败行 → 也按整批回滚（可重试），不冒险置 Done。
