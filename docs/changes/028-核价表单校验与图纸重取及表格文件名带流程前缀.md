- 状态：**已实现，待真机验证**
- 日期：2026-07-14

> 结论：本次三项优化——① 图纸核价「目标价」在**点击回传时**做格式校验（有效数字、大于 0、最多两位小数），
> 任一行非法则整批不回传；② 批次作业页操作列新增「重新获取图纸」按钮，可按料号重取一次图纸（同名覆盖 / 无则新增）；
> ③ 落盘的表格 Excel 文件名加上**作业流程名前缀**（如「图纸核价_<窗口>.xlsx」），并对历史无前缀命名做兼容与懒迁移。

## 一、目标价回传前校验

### 背景
核价表单的「目标价」是手填数值列（`FieldEditor.Number`、必填），此前无格式约束。行状态只按「非空」判定为已处理，
所以填了 `abc`、`10.555`、`-5` 也能通过提交闸门被回传出去。

### 改动
- 新增 `src/Draftline.Core/Validation/FieldRules.cs`：按字段 `Editor` 派发校验，数值列规则——
  - 必须是有效数字（拒 `abc`、`10a`、千分位 `1,000`、科学计数 `1e3`）；
  - 必须 **大于 0**（拒 `0`、负数）；
  - 小数位 **≤ 2**（拒 `10.555`）；
  - 空值放行（交由「必填」闸门处理）。
- 接入两处回传入口，**点击回传时、确认弹窗之前**校验：
  - `BatchWorkViewModel.Submit`（整批回传）：只校验将要回传的正常行（挂起异常行不回传、不拦）；
    **任一行非法 → 中止、按物料编码逐行提示，一次接口都不调**。
  - `ExceptionResolveViewModel.Upload`（异常单行补回传）：该行非法即中止。

### 说明
- 校验只针对「正常行」；挂起异常行不参与回传故不拦其目标价。
- `0` / 负数按「非法」处理（价格须 > 0）。若日后要允许 `0`，改 `FieldRules.ValidateNumber` 一处即可。
- 单元测试见 `tests/Draftline.Tests/Validation/FieldRulesTests.cs`。

## 二、批次作业页操作列新增「重新获取图纸」

### 背景
此前「重新获取图纸」只在异常补处理页（`ExceptionResolveViewModel`）有。批次作业页发现某料号缺图 / 图纸有更新时，
只能挂异常再去补处理页重取，绕路。

### 改动
- `BatchWorkViewModel` 新增按行命令 `RefetchDrawing(RowViewModel)`：复用已有网关接口 `/api/batch/refetch-drawing`——
  服务端向 PLM 重取该料号图纸落到服务器批次目录 → 客户端逐个下载并同步到本地批次目录、并入该行图纸清单、记操作日志。
- `BatchWorkView.xaml.cs` 操作列（`BuildActionColumn`）在「挂起异常/撤销异常」后加常驻「重新获取图纸」按钮（列宽 130→240）。

### 「同名覆盖 / 无则新增」语义
无需额外处理，本就是现有落盘语义：
- 服务器 `AppendDrawingsAsync`、本地 `WriteSyncFileAsync` 均用 `File.WriteAllBytesAsync` → 同名覆盖、异名新增；
- manifest 的 `AddRowDrawingsAsync` 按文件名（忽略大小写）去重 → 同名不重复登记、新名追加。

### 已知小项（未处理）
批次作业页有防抖自动暂存，`RefetchDrawing` 的 manifest 追加未走 `_saveLock`，理论上与自动暂存重写 manifest 有极小概率读改写竞争；
因内存图纸引用已同步、下次保存/扫盘可恢复，实际影响低，与异常补处理页现有做法一致。

## 三、表格 Excel 文件名带作业流程前缀

### 背景
落盘到服务器与本地的表格文件此前命名为 `<窗口>.xlsx`（= 批次目录名，纯数据范围），
文件离开目录后无法一眼区分属于哪个流程。

### 改动
- `LocalFolders.GridWorkbookName` 由 `<窗口>.xlsx` 改为 **`<流程名>_<窗口>.xlsx`**：
  - 图纸核价 → `图纸核价_20260519_1531-20260520_0930.xlsx`
  - 挑图 → `机加中心挑图_20260519_1531-20260520_0930.xlsx`
  - 前缀复用 `FlowFolder(flow)`（与顶层目录既有的流程中文名一致）。
- 服务器（`FileServerBatchStore`：ingest、`UpdateExcelRowAsync`）与本地（`LocalBatchStore`：ingest、`SaveBatchAsync`、两处读取）读写全部对齐；
  同步按实际文件名镜像，两侧统一后自动一致。

### 旧数据兼容 + 懒迁移
- 新增 `LegacyGridWorkbookName`（旧无前缀名）与 `ResolveGridWorkbookPath`（读取：优先新名，找不到回退旧名，只读判断不改盘）。
- 写入一律用新名，并顺手删掉同目录旧名 → **历史批次首次编辑/暂存即迁移到新名**，不会一个目录并存两个 xlsx；
  从不被编辑的历史批次仍可回退读取（保留旧名、不报错）。
- 未做上线时的一次性存量批量改名（当前为懒迁移）。若需要可另加启动迁移。
