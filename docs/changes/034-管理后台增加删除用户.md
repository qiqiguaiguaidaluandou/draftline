# 034 管理后台「用户管理」增加删除用户

- 状态：**实现完成，待运行态验证**
- 日期：2026-07-17
- 关联提交：`43e7245`（feat(admin): 管理后台用户管理增加删除用户（034））

## 需求

管理后台「用户管理」此前只能停用/启用用户（保留账号），无法真正删除用户。需要新增删除功能。

## 方案

在 `AdminService` 新增 `DeleteUserAsync`，同时被 REST 端点（`/api/admin/*`）与 Blazor 后台复用，逻辑不重复。删除为**硬删除**，与既有「停用」并存——停用保账号可复启用，删除彻底移除。

三重防护 + 数据处理：

- **不能删自己**：删掉当前登录管理员会立刻让本次会话失效，且易误操作（与「不能停用自己」同理）。
- **末位管理员保护**：删除比停用更彻底，只要删掉后不再有启用的管理员就拦下，避免没人能进后台。服务端兜底，绕过 UI 走 `/api/admin/*` 也拦得住。
- **手动清角色指派**：`UserRole` 仅对 `RoleId→Role` 建了外键级联，对工号（`EmployeeId`）没有外键，删用户不会自动带走其指派，须显式 `RemoveRange` 以免留孤儿行。
- **保留审计日志**：`ActivityLogs` 按工号存字符串、无外键，刻意保留——删人不抹历史。

## 实现

- `src/Draftline.Gateway/Auth/AdminService.cs`：接口 + 实现新增 `DeleteUserAsync`。
- `src/Draftline.Gateway/Endpoints/ApiEndpoints.cs`：新增 `DELETE /api/admin/users/{employeeId}`（与角色删除同风格）。
- `src/Draftline.Core/Logging/LogActions.cs` / `LogText.cs`：新增动作 `AdminDeleteUser`（中文名「删除用户」）。
- `src/Draftline.Gateway/Components/Pages/AdminUsers.razor`：每行加「删除」按钮，走 `ConfirmDialog` 二次确认（提示不可恢复、建议改用停用、日志保留）；删除的行若处于行内编辑态（配角色/重置密码）一并收起。
- `tests/Draftline.Tests/Auth/AdminServiceTests.cs`：新增 5 例——正常删除并清指派且留日志、删不存在、删自己、删末位管理员、有其他管理员时可删。
