## 现状一句话

PC 端（WPF）已是**完整 UI 骨架 + Mock 数据**，可编译、流程可跑通。
**后端无状态网关工程未起**，外部系统（EBS/PLM/SRM/DHR）接口规格未对齐，客户端真机验收未做。

契约接缝是干净的：真实 `Http*Gateway` 只要产出与 Mock 同样形状的 `FetchResult`/`SubmitResult`，
上层 UI、本地存储（`LocalBatchStore`）、状态机都不用改。切换点在 `DependencyInjection.cs`
（`AddDraftlineMockInfrastructure` → 新增 `AddDraftlineHttpInfrastructure`）。
